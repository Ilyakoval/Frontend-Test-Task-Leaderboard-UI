Frontend Test Task: Leaderboard UI
Опис проекту
Необхідно розробити веб-додаток для відображення рейтингової таблиці гравців у мобільній міні-грі "Гоночки". Суть гри — проходження дистанції на час.

Додаток орієнтований на мобільні пристрої, але має підтримувати адаптивність для екранів різного розміру. Дані про користувачів не потрібно отримувати з реального API — їх необхідно генерувати локально, імітуючи затримку мережі (лоадер).

Технічні вимоги
Стек: React (тільки функціональні компоненти) та React Hooks.
Адаптивність: Мінімальна ширина екрана — 320px, максимальна — 1920px.
Дані: Список користувачів генерується на стороні клієнта. Змінювати надані TypeScript-інтерфейси (див. розділ "Типи даних") заборонено.
Вимоги до функціоналу та продуктивності
Ініціалізація: При першому завантаженні відображаються перші 50 користувачів.
Infinite Scroll: Реалізувати lazy-load підвантаження (по 50 елементів), коли скрол наближається до нижньої межі екрана.
Продуктивність: Додаток має працювати плавно (без просідання FPS) навіть за наявності понад 1000 користувачів у списку.
Вимоги до UI/UX
Макет: Візуальна складова має бути максимально наближеною до наданого макету (поле "штрафний час" реалізовувати не потрібно).
Обрізка тексту: Якщо ім'я користувача не поміщається в доступну ширину, воно має обрізатися через три крапки (...).
Виділення: Клік по рядку користувача має виділяти його фіолетовим кольором.
Вирівнювання аватарок (Важливо): Аватарки всіх користувачів повинні утворювати єдину вертикальну лінію. Якщо при підвантаженні нових даних ця лінія зсувається праворуч (наприклад, через ширші порядкові номери), аватарки вже завантажених користувачів мають плавно (з анімацією) вирівнятися по новій осі.
Здача роботи
Викласти вихідний код проєкту на GitHub (публічний репозиторій).
Розгорнути додаток (наприклад, на Vercel, Netlify або GitHub Pages) та надати посилання для перегляду.

enum Color {
  RED,
